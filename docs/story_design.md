# BlockKing 剧情设计文档（拆分版）

更新时间：2026-02-15
状态：逐步拆分中（先锁定主线骨架，再补章节细节）

## 1. 文档目标与边界
- 本文档只维护剧情设计，不写玩法参数、不写美术调色细节。
- 章节采用四段式：`章节目标 / 关键事件 / 章节结尾信息 / 下一章钩子`。
- 玩法与素材信息继续保留在 `DESIGN.md`，剧情更新以本文为准。

## 2. 已锁定剧情基线
- 开场设定：主角受命执行“清剿小怪”的首次派遣任务。
- 主角能力：初始即具备攻击、物理格挡、魔法格挡。
- 教学策略：教学嵌入任务流程，不使用“装备/武器丢失”桥段。
- 第1章异常线索：箭矢尾羽金属箍编号异常。
- 第2章推进方式：上报被压下，随后私下追查。
- 第2章追查地点：废弃矿井仓储区。
- 第2章硬证据：运输账册残页。
- 怪物线到骑士线衔接：调查末尾出现高难怪物系 Boss 主动揭露。
- 第3章 Boss：`Orc rider`。
- 第3章揭露强度：仅透露“受人类骑士团资助/驱使”，不点具体姓名或代号。
- 明牌规则：第3章结尾点名骑士团，第4章开始正面对抗。

## 3. 六章剧情骨架（V0.1）

### 第1章：边境清剿
`章节目标`
- 完成首次派遣任务并在实战中自然完成双格挡教学。

`关键事件`
- 主角进入边境清剿区，按任务流程逐步处理低阶敌群。
- 清剿过程中出现异常箭矢，尾羽金属箍带有不应出现在该战区的编号。

`章节结尾信息`
- “普通清剿”被首次证据打断，主角确认战场来源存在异常。

`下一章钩子`
- 主角携带异常编号线索返回据点并发起上报。

### 第2章：矿井追查
`章节目标`
- 从“线索异常”推进到“组织层面异常”。

`关键事件`
- 上报线索后被压下，主角转为私下追查。
- 追查第一站为废弃矿井仓储区，沿编号路径定位可疑物资流。
- 在仓储深处取得运输账册残页。

`章节结尾信息`
- 账册显示异常编号并非偶发，背后存在稳定补给链路。

`下一章钩子`
- 顺着账册去向继续追查，进入更高危险区域。

### 第3章：追查失控
`章节目标`
- 将“异常补给链”升级为“阵营勾连”的明确信息。

`关键事件`
- 主角沿账册线索深入，遭遇更高强度怪物编成。
- 章节收尾战为 `Orc rider`，战斗压迫明显提升。
- Boss 在自认胜券在握时狂妄炫耀，主动透露其受骑士团资助/驱使。

`章节结尾信息`
- 骑士团与怪物并非单纯敌对关系，而是存在合作操盘。

`下一章钩子`
- 主角由暗查转向明面冲突，目标从“追线索”变为“对抗骑士团”。

### 第4章：明线对抗
`章节目标`
- 骑士团从暗线转为明牌，建立正面对抗主轴。

`关键事件`
- 主角进入骑士团控制区，遭遇组织化拦截与清除行动。
- 既有线索与现行行动互相印证，冲突从怀疑转为定性。

`章节结尾信息`
- 主角确认骑士团为当前阶段直接对立方，战争性质改变。

`下一章钩子`
- 转入更高层面的真相与代价问题：谁在骑士团之上，目的是什么。
